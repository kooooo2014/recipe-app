git の変更内容を確認してコミットを作成してください。

## 手順

1. `git status` と `git diff HEAD`（ステージ済み含む）を実行して変更を把握する
2. 変更の種類・目的をもとにコミットメッセージを日本語で作成する
3. 変更ファイルを適切にステージして `git commit` を実行する

## コミットメッセージのルール

- **1行目（件名）**: 変更の要約を50文字以内で。動詞から始める（例: `ホーム画面に検索機能を追加`, `買い物リストのバグを修正`）
- `$ARGUMENTS` が指定されていれば、その内容を件名のヒントとして優先的に使う
- 変更が複数の目的にまたがる場合は、最も主要な変更を件名にする
- 機密ファイル（.env など）は絶対にコミットしない

## ステージするファイルの判断

- 意図的な変更ファイルのみステージする（`git add -A` は使わず個別に `git add <file>`）
- ビルド成果物（`dist/`）、ログ、一時ファイルはステージしない

コミットが完了したら、作成したコミットメッセージと変更ファイルの一覧を報告してください。
