# レシピ帳アプリ

## 概要

スマートフォン向けのレシピ管理PWAアプリ。レシピの保存・閲覧・編集と、それに連動した買い物リスト機能を持つ。外部レシピサイトのURLからワンクリックでレシピをインポートできるのが特徴。

### 技術スタック

| 区分 | 技術 |
|------|------|
| UI | React 19 + TypeScript |
| スタイリング | Tailwind CSS v4 |
| ルーティング | React Router v7 |
| DB | Dexie (IndexedDB ラッパー) |
| ビルド | Vite 8 |
| PWA | vite-plugin-pwa (Workbox) |
| アイコン | lucide-react |

データはすべてブラウザのIndexedDBに保存されるため、サーバーレスで動作する。レシピインポートAPIは Vercel Edge Function（`api/fetch-recipe.ts`）として実装済み。

## できていること

### レシピ管理

- **一覧表示**（`/`）— サムネイル・料理名・調理時間をカード形式で表示、タイトルのインクリメンタル検索付き
- **詳細表示**（`/recipe/:id`）— 画像ヘッダー、説明、材料、作り方の手順番号付きリスト、元レシピへのリンク
- **レシピ追加**（`/add`）— タイトル・説明・画像URL・人数・調理時間・材料・手順を入力して保存
- **レシピ編集**（`/recipe/:id/edit`）— 既存レシピの全フィールドを編集・上書き保存
- **レシピ削除** — 詳細画面からワンタップで削除（確認ダイアログあり）

### URLインポート

対応サイトのURLを貼り付けるだけで、ページの`application/ld+json`（Recipe構造化データ）を解析してフォームへ自動入力する。

対応サイト: クックパッド / 楽天レシピ / デリッシュキッチン / クラシル / マカロニ

- 分数（`1/2`）や「大さじ」「少々」など日本語の分量表記を解析して材料に変換
- ISO 8601 形式の調理時間（`PT30M`）をパース

### 人数調整

- 詳細画面で人数を ＋／－ で増減すると、材料の分量がリアルタイムで比例計算される
- 「少々」「適量」など比率換算しない単位は除外して表示

### 買い物リスト

- 詳細画面の「買い物リストに追加」ボタンで現在人数分の材料を一括追加（`/shopping`）
- 手動で品目を追加することも可能
- チェックボックスでON/OFF、チェック済み一括削除
- レシピごとにグルーピング表示

### タイマー

- 詳細画面に調理時間のカウントダウンタイマーを内蔵
- スタート/一時停止/リセット、プログレスバー表示
- 時間を任意に変更可能
- タイマー終了時にブラウザ通知（Notification API）を送信

### PWA対応

- スマートフォンのホーム画面にインストール可能（スタンドアロン表示）
- Workbox によるオフラインキャッシュ

## これからしたいこと

### 優先度：高

- [x] **本番用レシピインポートAPI** — `api/fetch-recipe.ts` を Vercel Edge Function として実装。`vite.config.ts` のミドルウェアはローカル開発用として継続利用
- [ ] **画像ローカル保存** — 現在は画像URLを文字列保存するだけなので、URLが切れると画像が消える。ファイル選択でBase64やBlob URLに変換してIndexedDBへ保存する

### 優先度：中

- [ ] **タグ・カテゴリ機能** — レシピにタグを付けて絞り込みできるようにする
- [ ] **並び替え・フィルタ** — 料理時間順、登録日順などの切り替え
- [ ] **レシピのエクスポート/インポート** — JSON形式でバックアップ・他端末への移行
- [ ] **買い物リストの統合** — 同じ材料が複数レシピに含まれる場合に分量をまとめる

### 優先度：低

- [ ] **ダークモード** — Tailwind の`dark:`バリアントで対応
- [ ] **レシピ共有** — URL共有や QRコード生成
- [ ] **栄養素表示** — 材料データと外部APIを組み合わせてカロリー等を計算
